# rustcluster.index v1.2 — Execution Plan

> Companion to `docs/v1.2-plan-pure-rust-simd.md` (the research/context doc).
> This is the actionable layer on top: decisions locked in, sequencing,
> decision gates, acceptance criteria.

## Decisions locked in

- **Approach:** Pure-Rust SIMD via `pulp` (already a transitive dep through
  faer 0.24, version 0.22.2). Falls back to raw `std::arch` if Step 0a
  invalidates pulp.
- **Load-bearing optimization:** the fused threshold-emit kernel for
  `similarity_graph`. Microkernel quality is secondary.
- **Architectures shipped in v1.2:** NEON (aarch64) is the primary tuning
  target. AVX2 (x86_64) is also CI-tested. AVX-512 is best-effort,
  runtime-detected, manual validation only — not gated on CI.
- **Primary benchmarking machine:** Apple Silicon (M-series). All
  performance targets in this plan are M-series numbers.
- **Crate publishing:** `rustcluster-simd` stays path-dep-only for v1.2.
  Reconsider crates.io publication after v1.2 has shipped one release
  cycle. Reasoning: the fused threshold-emit kernel signature is the most
  uncertain piece, and publishing prematurely commits to semver.
- **Out of scope for v1.2:** Apple AMX, ARM SVE/SVE2, BF16/int8, new
  public API.

## Sequencing — Phase 0: Decision week (~1 week)

The whole point of Phase 0 is to spend a week of measurement before
committing to 2-3 months of build.

### Step 0a — Pulp microkernel prototype (1-2 days)

`examples/probe_pulp_dot.rs` — implement f32 dot product four ways and
compare:

1. Scalar reference (correctness baseline).
2. Pulp via `WithSimd` (the candidate primary path).
3. Raw `std::arch` AVX2+FMA (x86_64) — lower bound for "what hand-tuning
   could deliver."
4. Raw `std::arch` NEON (aarch64) — same.
5. faer matmul shaped as `1×d × d×1` (current production path).

Bench at d ∈ {128, 384, 1536} on Apple Silicon.

**Decision thresholds:**

| Pulp vs std::arch dot at d=1536 | Action |
|---|---|
| within 15% | Pulp is primary. Proceed with Plan A. |
| 15-30% behind | Pulp is primary, plan a NEON hot-path variant for v1.3. |
| >30% behind | Fall back to raw `std::arch` for v1.2. Plan B. |

PR #1 / PR #2 status does not block this step — the probe is
purely additive (`examples/probe_pulp_dot.rs`, no edits to existing code).

### Step 0b — Fused-kernel architectural spike (2-3 days)

Write a *temporary* `fused_threshold_emit_tile` that uses **faer matmul as
the inner compute** (no new microkernel). Wire it into
`similarity_graph_ip` behind a feature flag. Bench at d=1536, n=20k,
threshold=0.5 against the existing path.

**Question this answers:** how much of the FAISS gap does fusion alone
close? If fusion + faer reaches ≥0.95× FAISS, the microkernel work in
Plan A is bonus, not load-bearing.

This step touches `src/index/similarity_graph.rs` so it should wait until
PR #2 is at least review-stable to avoid rebase pain.

### Step 0c — Decision gate

Document both measurements in `docs/v1.2-prototype-result.md` (created in
this step). Pick one:

**Resolved (2026-05-01).** Step 0a validated pulp ×4 (within 2.7% of
std::arch NEON at d=1536). Step 0b showed that fusion-with-faer-inner
is *slower* than baseline at every sub-tile size — the microkernel
work is load-bearing, not bonus.

| Outcome of 0a + 0b | Plan | Effort | Status |
|---|---|---|---|
| Pulp validates AND microkernel is load-bearing | **Plan A** — pulp + fused kernel | 7-9 weeks | **Selected** |
| Pulp fails AND microkernel is load-bearing | Plan B — std::arch + fused kernel | 16-20 weeks | Ruled out (pulp validated) |
| Fusion alone closes the gap (any pulp result) | Plan C — fused kernel only, faer inner | 3-4 weeks | Ruled out (Step 0b) |
| Pulp validates AND fusion alone closes the gap | Plan C+ — fused kernel + opportunistic pulp microkernel | 5-7 weeks | Ruled out (Step 0b) |

### Milestone 3 — `l2sq_f32` + tests (2 days)

Identity-derived from dot. proptest vs scalar.

### Milestone 4 — `batched_dot_tile` (1 week)

Tile-shaped microkernel:

```rust
pub fn batched_dot_tile(
    q: &[f32], q_h: usize,
    x: &[f32], x_w: usize,
    d: usize,
    out: &mut [f32],
);
```

Bench against current `faer::matmul` at tile sizes {96, 256, 384}.

**Acceptance:** within 15-30% of faer at d=1536; proptest green.

### Milestone 5 — Wire `batched_dot_tile` into `ip_batch` (2 days)

Replace the `faer::matmul` call in `src/index/kernel.rs::ip_batch`
(lines 64-81). Keep faer fallback behind `#[cfg(feature = "faer-fallback")]`
for one release.

**Acceptance:** all 359 Python + 241 Rust tests green; bench at d=1536
within 5% of v1.1.

### Milestone 6 — `fused_threshold_emit_tile` proper (2 weeks)

Write the production fused kernel using `batched_dot_tile`-style
register blocking with the threshold check inlined into the FMA
accumulation loop. Avoid materializing the score tile.

**Note from Step 0b:** the faer-backed sub-tile spike (now removed
from `similarity_graph.rs`) was *slower* than baseline at every
sub-tile size. The architectural win requires the threshold check
to live inside the register-blocked accumulator loop — external
sub-tiling with faer-inner-compute is a dead end. Go straight to
the pulp ×4 inner kernel with fused threshold check; do not build
a faer-backed intermediate.

**Acceptance:** edge-set parity test green vs `batched_dot + threshold
filter`; bench shows ≥1.0× FAISS at d=1536.

### Milestone 7 — Wire fused kernel into `similarity_graph_l2/_ip` (2-3 days)

Replace the `ip_batch + nested loop` pattern in
`src/index/similarity_graph.rs`:
- L2 path: lines 99-127.
- IP path: lines 146-172.

Keep the existing path behind a feature flag for one release.

**Acceptance:** all `test_index_similarity_graph.py` tests green;
brute-force-parity tests still pass.

### Milestone 8 — Per-arch tuning, NEON priority (1 week, optional)

Hand-tune the inner microkernel for NEON if Step 0a / Milestone 4 bench

### Milestone 9 — Bench sweep + perf-marker test updates (3 days)

Run `benches/index.rs` at all v1.1 shapes. Update

## Performance targets (n=20k, d=1536, threshold=0.5, Apple Silicon)

| Operation | Current (v1.1) | Minimum acceptable | Stretch |
|---|---|---|---|
| `similarity_graph` d=128 | 2.32× FAISS | ≥2.0× | ≥2.0× |
| `similarity_graph` d=1536 | 0.77× FAISS | ≥1.0× | ≥1.2× |
| `search` d=1536 | 0.32× FAISS | ≥0.5× | ≥0.7× |
| `range_search` d=1536 | 0.37× FAISS | ≥0.5× | ≥0.7× |

**Hard gate:** no regression at d=128 anywhere.

## Risk register

| Risk | Trigger | Action |
|---|---|---|
| Pulp >30% behind std::arch | Step 0a measurement | Switch to Plan B; calendar grows to 16-20 weeks. Notify before proceeding. |
| Fusion alone doesn't close the gap and pulp is also weak | Step 0a + 0b both negative | Re-scope: ship `range_search`/`search` improvements only, defer `similarity_graph` win. Discuss. |
| Milestone 6 (fused kernel) overruns by >2 weeks | Mid-build | Escalate. Architecture might be wrong; try alternative blocking. |
| AVX-512 silently miscomputes in production | Manual-only validation | Add `RUSTCLUSTER_DISABLE_AVX512=1` runtime escape hatch. Document it. |
| Wheel CI breaks on aarch64 | First aarch64 CI run | Add `ubuntu-22.04-arm64` runner before milestone 1. Validate CI shape before kernel work. |
| PR #1/#2 review takes longer than expected | Stalled on review | Step 0a can run in parallel; only Step 0b and Milestone 1+ block on PR #2 review-stability. |
